Texto Original do Comunicado à Imprensa

1 de Julho de 2009 O Grupo de Desenvolvimento Global do PostgreSQL lançou a versão 8.4, continuando o rápido desenvolvimento do banco de dados de código aberto mais avançado do mundo. Esta versão contém várias melhorias para tornar a administração, consulta e programação de bancos de dados PostgreSQL mais fáceis do que nunca. Com 293 modificações entre novos recursos e melhorias na versão 8.4, existem agora mais motivos para escolher o PostgreSQL para o seu próximo projeto.

As mudanças mais numerosas no PostgreSQL 8.4 são ferramentas e comandos para administração e monitoramento novos ou melhorados. Cada usuário tem sua funcionalidade favorita que tornará o seu dia-a-dia com o PostgreSQL mais fácil e produtivo.

"Nós usamos o PostgreSQL há 7 anos e estamos aguardando muitos dos recursos da 8.4, em particular a permissão de colunas, a configuração regional (locales) por banco de dados, buscas parciais em índices GIN e exceções definidas pelo usuário," disse Jeffrey Webster, CTO da ZooLoo.com. "PostgreSQL nos permitiu crescer sem sacrificar a integridade dos dados."

Entre as melhorias mais populares estão:

  • Restauração Paralela de Bases de Dados, aumentando a velocidade de recuperação de cópia de segurança (backup) em até 8 vezes
  • Permissões por Coluna, permitindo um controle mais granular de dados sigilosos
  • Suporte a Configuração Regional por Banco de Dados, tornando o PostgreSQL mais útil em ambientes com múltiplos idiomas
  • Upgrades In-place com o pg_migrator beta, permitindo migrações da 8.3 para a 8.4 sem tirar o banco do ar por muito tempo
  • Novas Ferramentas de Monitoramento de Consultas, dando aos administradores mais detalhes sobre a atividade das consultas

A versão 8.4 facilita a análise de dados implementando funcionalidades avançadas do ANSI SQL2003 como windowing functions, common table expressions e junções recursivas. "Essas estruturas de consulta aumentam substancialmente a expressividade do dialeto SQL do PostgreSQL, permitindo aos usuários formularem perguntas interessantes em uma única consulta que seria impossível formular anteriormente," explica Sailesh Krishnamurthy, Fundador da Truviso. Melhorias nas funções armazenadas (stored procedures), como parâmetros padrão e listas de parâmetros variáveis, tornam a programação no servidor de banco de dados mais simples e compacta.

A nova versão também aumenta a performance de aplicações, como comenta Kevin Grittner, Administrador de Bancos de Dados para o Sistema Judicial de Wisconsin (EUA), "O PostgreSQL continua a melhorar performance em cada nova versão. A versão 8.4 adiciona muitas otimizações, tais como semi-junções e anti-junções, que trazem melhorias dramáticas em tempo de execução para nossas consultas mais exigentes."

Essas funcionalidades significam que o PostgreSQL 8.4 servirá mais usuários que nunca, tal como o projeto OpenStreetMap. "Quando estávamos planejando a nova versão da API do OpenStreetMap, ficou claro que precisaríamos de um banco de dados de classe mundial que não só traz as funcionalidades certas, mas também roda bem com a capacidade de escalonar que precisamos. Apesar de haver muitos bancos de dados disponíveis, o PostgreSQL foi a escolha óbvia," segundo Tom Hughes, administrador de sistemas do OpenStreetMap.

Licença

O PostgreSQL usa a licença BSD, que requer apenas que o código fonte licenciado mantenha seu direito de cópia e informação da licença. Esta licença certificada pela OSI é amplamente vista como flexível e amigável à empresas, já que ela não restringe o uso do PostgreSQL com aplicações comerciais e proprietárias. Juntamente com suporte de múltiplas empresas e propriedade pública do código, a licença BSD torna o PostgreSQL muito popular com fabricantes que querem anexar um banco de dados com seus próprios produtos sem medo de ônus financeiro, prisão à um só fabricante, ou mudanças nos termos de licenças.
